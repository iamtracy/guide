Technical Requirements, System Checks and Network Preparation

A strong internet connection and a computer that meets the required specifications for delivery will be important to ensuring a successful webcast.

Speaker Computer Requirements:

  • Each computer that is logging in to the XPOCAST Presenter Portal as either speaker or support staffer should meet the following requirements to ensure proper connectivity to the XPOCAST system.

  • Have a Hard Wired Internet Connection: Turn off wireless access on the computer to ensure the computer is using the hardwired connection.

  • Pass the System Check: Speakers must pass on each item in the system check to properly present: https://vts.inxpo.com/Launch/Event.htm?

Network Requirements:

  • When giving a webcast, particularly a video webcast, it is important to ensure the speaker’s network is able to handle streaming to the internet and will not impede the delivery or cause an issue for the speaker and/or the audience.

  • Speakers should confirm that their presentation location has enough available bandwidth for an XPOCAST presentation in advance of the dry run. A minimum of 600 KBPS “up” is recommended for delivering a presentation.

To calculate available bandwidth:

  1. Go to the speed test and start the test (select Chicago)
  2. Take the “Upload Speed” result (given in MBPS)
  3. Access a MBPS to KBPS converter (i.e. Check Your Math) and enter your MBPS number and convert to KBPS

In addition users should pass on each of the Port Tests.

Tips and Tricks For Delivery:

When planning to present using XPOCAST it is recommended that each speaker:

  • Test their system as soon as they sign on to speak
  • Determine their delivery location and test the network at least 48 hours before the dry run
  • Participate in a dry run of the webcast to ensure connectivity with their engineer
  • Always use the same location/equipment used for testing on the live day
